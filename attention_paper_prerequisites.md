> 这篇论文（2017, Vaswani et al.）提出了 Transformer 架构，是 GPT、BERT 等所有现代大模型的基石。
> 下面按**从基础到进阶**的顺序，列出你需要掌握的知识点。

## 一、线性代数（必须掌握）

这是论文里每一页都在用的数学语言。

| 概念 | 你需要理解到什么程度 | 论文中哪里用到 |
|------|---------------------|---------------|
| **向量** | 一组有序的数字，表示一个"方向+大小" | Token Embedding 就是向量 |
| **矩阵** | 二维数字表格，可以理解为一组向量或一个线性变换 | 权重矩阵 W^Q, W^K, W^V |
| **矩阵乘法** | A(m×n) × B(n×p) = C(m×p)，以及它的物理含义——线性变换/投影 | Attention 计算的每一步 |
| **转置 (Transpose)** | 矩阵的行列互换，记作 Kᵀ | Q·Kᵀ 计算注意力分数 |
| **点积 (Dot Product)** | 两个向量逐元素相乘再求和，衡量"相似度" | Scaled Dot-Product Attention 的核心 |
| **向量的模 / 范数** | ‖v‖ = √(v₁² + v₂² + ...) | 理解为什么要除以 √d_k |

### 自测：你能手算这个吗？

```
Q = [[1, 0],     K = [[1, 1],
     [0, 1]]          [0, 1]]

Q · Kᵀ = ?
```

答案：`[[1, 0], [1, 1]]`。如果你能算出来，线性代数部分就够了。

## 二、概率与统计（必须掌握）

| 概念 | 说明 | 论文中哪里用到 |
|------|------|---------------|
| **概率分布** | 一组非负数，加起来等于 1 | Attention 权重就是概率分布 |
| **Softmax 函数** | 把任意实数向量变成概率分布：softmax(x_i) = e^{x_i} / Σe^{x_j} | 论文公式 (1) 的核心 |
| **交叉熵损失 (Cross-Entropy)** | 衡量预测概率分布与真实分布的差距 | 训练目标函数 |
| **条件概率** | P(下一个词 \| 前面的词) | 自回归语言模型的基础 |

## 三、深度学习基础（必须掌握）

| 概念 | 说明 | 论文中哪里用到 |
|------|------|---------------|
| **神经网络** | 由多层线性变换 + 非线性激活函数组成 | 整个 Transformer 就是一个深度神经网络 |

## 四、序列建模背景（了解即可）

论文的核心论点是 "Attention 就够了，不需要 RNN/CNN"，所以你需要了解它在**替代什么**。

| 概念 | 你需要知道的 | 为什么论文要提它 |
|------|-------------|-----------------|
| **RNN / LSTM** | 按时间步顺序处理序列，t 时刻依赖 t-1 时刻的输出 | Transformer 的对手——RNN 无法并行，长距离依赖弱 |
| **Encoder-Decoder 架构** | Encoder 理解输入，Decoder 生成输出 | Transformer 沿用了这个大框架 |
| **Seq2Seq** | 序列到序列的任务，如机器翻译 | 论文用机器翻译作为主要实验任务 |

> 你不需要会实现 RNN，只需理解它的**缺陷**：顺序处理慢、长距离信息容易遗忘。
> 这样你才能体会 Transformer 的设计动机。

## 六、论文中的核心新概念（读论文时重点攻克）

以下是论文**提出或强调**的概念，读到时需要重点理解：

### 6.1 Self-Attention（自注意力）

```
核心公式：Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V

拆解：
  Q (Query)  → "我在找什么？"
  K (Key)    → "我能提供什么？"
  V (Value)  → "我的实际内容是什么？"

  QKᵀ        → 每对 (query, key) 之间的相似度分数
  / √d_k     → 缩放，防止点积值过大导致 softmax 梯度消失
  softmax    → 转成概率（注意力权重）
  · V        → 用注意力权重对 value 加权求和
```

### 6.2 Multi-Head Attention（多头注意力）

```
不是做一次 Attention，而是做 h 次（h=8），每次用不同的 W^Q, W^K, W^V

好处：每个"头"可以关注不同的语义关系
  - 头1 可能关注"语法结构"（主语在哪？）
  - 头2 可能关注"指代关系"（"它"指什么？）
  - 头3 可能关注"位置邻近"（前一个词是什么？）

最后把 h 个头的结果拼接起来 → 线性变换 → 输出
```

### 6.3 Positional Encoding（位置编码）

```
问题：Attention 是"集合运算"，不区分词序
     → "猫吃鱼" 和 "鱼吃猫" 在 Attention 看来一样！

解决：给每个位置加上一个独特的位置向量
     论文使用 sin/cos 函数生成：
       PE(pos, 2i)   = sin(pos / 10000^(2i/d))
       PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

### 6.4 残差连接 + 层归一化 (Add & Norm)

```
output = LayerNorm(x + SubLayer(x))
                   ↑
                   残差连接：把输入直接加回来

为什么需要？
  - 残差连接：让梯度能"直通"，训练深层网络不卡住
  - 层归一化：稳定每层的数值范围，加速训练
```

### 6.5 Masked Attention（掩码注意力）

```
在 Decoder 中，生成第 t 个词时不能"偷看"后面的词

做法：把未来位置的注意力分数设为 -∞
     → softmax(-∞) = 0
     → 等于完全屏蔽了未来信息
```
